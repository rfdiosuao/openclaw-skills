## 1. Skill 简介

### 1.1 什么是 OpenClaw Security Guard？

**OpenClaw Security Guard** 是一款基于 OpenClaw 官方安全文档开发的专业安全检测工具，能够自动扫描 16 项核心安全风险，及时发现配置问题并提供修复建议。

### 1.2 核心价值

✅ **全面检测** - 覆盖文件权限、网络暴露、认证授权、工具执行、沙箱隔离等  
✅ **智能告警** - 严重/警告/信息三级分类，优先处理关键问题  
✅ **一键修复** - 自动修复常见问题，提供详细修复命令  
✅ **定时巡检** - 支持心跳任务，每日自动检测  
✅ **主动通知** - 发现严重风险时立即通知管理员  

### 1.3 检测范围

| 类别 | 检测项数 | 说明 |
|------|---------|------|
| 文件系统安全 | 2 | 状态目录权限、配置文件权限 |
| Gateway 安全 | 3 | 认证配置、绑定配置、Tailscale |
| 沙箱隔离 | 2 | 网络模式、启用状态 |
| 工具策略 | 3 | Exec 安全、Elevated 模式、权限配置 |
| 会话管理 | 1 | DM 隔离 |
| 日志安全 | 1 | 脱敏配置 |
| 插件安全 | 1 | 允许列表 |
| 浏览器安全 | 1 | SSRF 策略 |
| 节点安全 | 1 | 命令限制 |
| **合计** | **16** | **全面覆盖** |

## 2. 快速开始

### 2.1 安装 Skill

#### 方式一：通过 Claw-CLI 安装（推荐）

```bash
claw skill install openclaw-cn/openclaw-security-guard
```

#### 方式二：手动安装

1. 下载 Skill 包（`openclaw-security-guard.tar.gz`）
2. 解压到 Skills 目录
3. 安装依赖并编译

```bash
# 解压
tar -xzf openclaw-security-guard.tar.gz -C ~/.openclaw/skills/

# 进入目录
cd ~/.openclaw/skills/openclaw-security-guard

# 安装依赖
npm install

# 编译
npm run build
```

### 2.2 首次使用

```bash
# 执行全面检测
security-guard audit

# 查看检测结果
# 系统会输出详细的检测报告
```

### 2.3 典型输出

```

═══════════════════════════════════════

📊 检测概览
```

## 3. 功能详解

### 3.1 全面审计

**命令：** `security-guard audit`

**功能：** 执行 16 项全面安全检测

**参数：**
- `--deep` - 深度审计（包含实时探测）
- `--json` - JSON 格式输出
- `--quiet` - 静默模式，仅输出结果

**示例：**
```bash
# 标准审计
security-guard audit

# 深度审计
security-guard audit --deep

# JSON 输出
security-guard audit --json > report.json
```

### 3.2 快速检查

**命令：** `security-guard quick-check`

**功能：** 仅检查关键严重风险（5 项）

**适用场景：** 日常快速验证配置安全性

**示例：**
```bash
security-guard quick-check
```

### 3.3 专项检查

**命令：** `security-guard check <category>`

**支持的类别：**
- `fs-perms` - 文件权限检查
- `network` - 网络配置检查
- `auth` - 认证配置检查
- `sandbox` - 沙箱配置检查
- `tools` - 工具策略检查

**示例：**
```bash
# 检查文件权限
security-guard check fs-perms

# 检查网络配置
security-guard check network

# 检查认证配置
security-guard check auth
```

### 3.4 自动修复

**命令：** `security-guard fix`

**功能：** 自动修复可修复的安全问题

**参数：**
- `--all` - 修复所有可修复的问题
- `--critical` - 仅修复严重问题
- `--dry-run` - 模拟运行，不实际修改

**示例：**
```bash
# 修复所有问题
security-guard fix --all

# 仅修复严重问题
security-guard fix --critical

# 模拟运行（查看将执行的操作）
security-guard fix --dry-run
```

### 3.5 生成报告

**命令：** `security-guard report`

**支持的格式：**
- `text` - 文本格式（默认）
- `json` - JSON 格式
- `markdown` - Markdown 格式

**示例：**
```bash
# 生成文本报告
security-guard report

# 生成 JSON 报告
security-guard report --format json -o report.json

# 生成 Markdown 报告
security-guard report --format markdown -o report.md
```

### 3.6 告警管理

**命令：** `security-guard alert`

**功能：** 配置安全告警通知

**子命令：**
- `config` - 配置告警渠道
- `test` - 测试告警
- `history` - 查看告警历史

**示例：**
```bash
# 配置飞书告警
security-guard alert config --channel feishu --user ou_xxx

# 测试告警
security-guard alert test

# 查看告警历史
security-guard alert history
```

## 4. 使用场景

### 场景一：新环境部署检查

**背景：** 刚安装 OpenClaw，需要确保配置安全

**操作步骤：**

```bash
# 1. 执行全面检测
security-guard audit

# 2. 查看检测结果
# 识别所有严重和警告风险

# 3. 自动修复
security-guard fix --all

# 4. 验证修复结果
security-guard quick-check
```

**预期结果：** 所有严重风险已修复，警告风险已处理

---

### 场景二：配置变更后验证

**背景：** 修改了 OpenClaw 配置文件，需要验证安全性

**操作步骤：**

```bash
# 1. 修改配置文件后
# 编辑 ~/.openclaw/openclaw.json

# 2. 执行专项检查
security-guard check auth      # 检查认证
security-guard check network   # 检查网络
security-guard check sandbox   # 检查沙箱

# 3. 快速验证
security-guard quick-check
```

**预期结果：** 配置变更未引入新的安全风险

---

### 场景三：每日安全巡检

**背景：** 定期检查 OpenClaw 安全状态

**配置心跳任务：**

在 `HEARTBEAT.md` 中添加：

```markdown
### 安全检查（每日 02:00）
- [ ] **执行安全审计** - `security-guard audit --quiet`
- [ ] **检查严重风险** - 发现 critical 立即告警
- [ ] **生成日报** - 保存检测报告
```

**自动化脚本：**

```bash
#!/bin/bash
# 每日安全检查脚本

DATE=$(date +%Y-%m-%d)
REPORT_DIR="/var/log/security-audit"

# 执行审计
security-guard audit --json > "$REPORT_DIR/report-$DATE.json"

# 检查严重风险
CRITICAL=$(cat "$REPORT_DIR/report-$DATE.json" | jq '.summary.critical')

if [ "$CRITICAL" -gt 0 ]; then
    # 发送告警
    security-guard alert test
fi
```

### 场景四：生成安全报告

**背景：** 需要向管理层汇报安全状态

**背景：** 发现可疑活动，需要紧急安全检查

**操作步骤：**


## 5. 配置指南

### 5.1 最小安全配置

以下是推荐的 OpenClaw 最小安全配置：

```json
{
  "gateway": {
    "mode": "local",
    "bind": "loopback",
    "port": 18789,
    "auth": {
      "mode": "token",
      "token": "your-long-random-token"
    }
  },
  "session": {
    "dmScope": "per-channel-peer"
  },
  "tools": {
    "profile": "messaging",
    "deny": ["group:automation", "group:runtime", "group:fs"],
    "fs": { "workspaceOnly": true },
    "exec": {
      "security": "deny",
      "ask": "always"
    },
    "elevated": { "enabled": false }
  },
  "agents": {
    "defaults": {
      "sandbox": { "mode": "non-main" }
    }
  },
  "logging": {
    "redactSensitive": true
  }
}
```

### 5.2 告警配置

**配置飞书告警：**

```bash
security-guard alert config --channel feishu --user ou_9822a6c7f8a0ee947a95b0f212c6a85e
```

**配置 Telegram 告警：**

```bash
security-guard alert config --channel telegram --user <chat_id>
```

### 5.3 定时任务配置

**Cron 配置示例：**

```bash
# 编辑 crontab
crontab -e

# 添加每日检查（每天 02:00）
0 2 * * * /usr/bin/security-guard audit --quiet >> /var/log/security-audit.log 2>&1

# 添加每周报告（每周日 09:00）
0 9 * * 0 /usr/bin/security-guard report --format markdown -o /var/log/security-audit/weekly-$(date +\%Y-\%m-\%d).md
```

## 6. 常见问题

### Q1: 命令未找到

**问题：** 运行 `security-guard` 提示命令不存在

**解决方案：**

```bash
# 确保已安装
npm install -g @openclaw-cn/security-guard

# 检查 PATH
echo $PATH

# 重新链接
npm link
```

---

### Q2: 无法访问配置文件

**问题：** 检测时提示无法访问配置文件

**解决方案：**

```bash
# 检查文件是否存在
ls -la ~/.openclaw/openclaw.json

# 检查权限
chmod 600 ~/.openclaw/openclaw.json

# 检查 OpenClaw 是否安装
openclaw status
```

---

### Q3: 检测失败

**问题：** 某项检测执行失败

**解决方案：**

```bash
# 查看详细错误
security-guard audit --verbose

# 检查 OpenClaw 安装
openclaw doctor

# 查看日志
tail -100 ~/.openclaw/logs/gateway.log
```

---

### Q4: 自动修复失败

**问题：** `security-guard fix` 执行失败

**解决方案：**

```bash
# 使用 dry-run 查看将执行的操作
security-guard fix --dry-run

# 手动执行修复命令
# 根据输出手动执行相应的 chmod 或其他命令

# 检查权限
whoami
id
```

---

### Q5: 告警未发送

**问题：** 配置了告警但未收到通知

**解决方案：**

```bash
# 测试告警
security-guard alert test

# 检查告警配置
cat ~/.openclaw/security-guard/alert-config.json

# 检查告警历史
security-guard alert history
```

## 7. 附录

### 7.1 检测项详细说明

#### 严重风险（Critical）

| 检测项 ID | 说明 | 风险 | 修复方法 |
|----------|------|------|----------|
| `fs.state_dir.perms_world_writable` | 状态目录可写 | 其他用户可修改 OpenClaw 状态 | `chmod 700 ~/.openclaw` |
| `fs.config.perms_writable` | 配置文件可写 | 攻击者可能篡改认证配置 | `chmod 600 ~/.openclaw/openclaw.json` |
| `gateway.auth_missing` | Gateway 无认证 | 任何人都可以控制你的 OpenClaw | 配置 `gateway.auth.mode: "token"` |
| `gateway.bind_no_auth` | Gateway 暴露无认证 | 未授权访问 Gateway API | 绑定 loopback 或启用认证 |
| `gateway.tailscale_funnel` | Tailscale Funnel 暴露 | Gateway 暴露到公网 | 禁用 Funnel 模式 |
| `sandbox.dangerous_network_mode` | 沙箱网络模式危险 | 沙箱可访问主机网络 | 设置 `network: "none"` |

#### 警告风险（Warning）

| 检测项 ID | 说明 | 风险 | 修复方法 |
|----------|------|------|----------|
| `tools.exec.host_sandbox_no_sandbox` | Exec 无沙箱 | 可执行任意命令 | 设置 `security: "ask"` |
| `tools.elevated.enabled` | Elevated 模式启用 | 绕过沙箱执行 | 设置 `enabled: false` |
| `sandbox.mode_off` | 沙箱未启用 | 工具直接在主机执行 | 设置 `mode: "non-main"` |
| `logging.redact_off` | 日志未脱敏 | 敏感信息泄露 | 启用 `redactSensitive: true` |
